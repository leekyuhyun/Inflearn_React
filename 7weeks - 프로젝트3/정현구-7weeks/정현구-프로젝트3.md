# 📒 [Javascript] 7주차 감정 일기장 프로젝트 (Emotional Diary)

React와 Vite를 기반으로 제작된 개인 맞춤형 감정 일기장 프로젝트 
하루의 기록을 남기고, 그때의 감정을 5단계의 이미지로 저장

[🚀 배포 링크: 감정 일기장](https://emotion-dinary.vercel.app/)

## 주요 기능

1. **일기 목록 (Home)**
   - 연도/월별 필터링 기능
   - 감정별(최신순/오래된순) 정렬 기능
2. **일기 작성 (New)**
   - 날짜 선택 및 감정 5단계 선택
   - 텍스트 기반 일기 내용 작성
3. **일기 상세 (Diary)**
   - 작성된 일기의 감정, 내용, 날짜 확인
4. **일기 수정 및 삭제 (Edit)**
   - 기존 데이터 불러오기 및 수정 처리
   - 데이터 영구 삭제 기능
5. **데이터 영속성**
   - 브라우저의 LocalStorage를 활용하여 새로고침 후에도 유지되는 데이터

## 📂 프로젝트 구조

```text
src/
├── assets/      # 감정 아이콘 (1~5번)
├── components/  # 공통 컴포넌트 (Button, Header, Editor 등)
├── hooks/       # 커스텀 훅 (useDiary, usePageTitle)
├── pages/       # 페이지 단위 컴포넌트 (Home, New, Diary, Edit)
├── util/        # 공통 로직 및 상수 (get-emotion-image 등)
├── App.jsx      # 전역 상태(Context) 및 라우팅 설정
└── main.jsx     # 앱 엔트리 포인트
```

## 세부사항

### 1. 전역 상태 관리 (`useReducer` + `Context API`)
`App.jsx`에서 `useReducer`를 사용해 일기의 **생성(CREATE)**, **수정(UPDATE)**, **삭제(DELETE)** 로직을 중앙 집중화했습니다. 또한 `DiaryStateContext`와 `DiaryDispatchContext`를 분리하여 불필요한 리렌더링을 방지했습니다.

### 2. 효율적인 데이터 상세 조회 (`useDiary` Hook)
많은 페이지에서 특정 ID의 일기 데이터를 필요로 하므로, `useDiary`라는 커스텀 훅을 만들어 데이터 필터링 및 예외 처리(존재하지 않는 페이지)를 통합 관리했습니다.

### 3. 사용자 경험(UX) 최적화
- **동적 페이지 타이틀**: `usePageTitle` 훅을 통해 각 페이지 진입 시 브라우저 탭의 제목을 동적으로 변경합니다.
- **날짜 포맷팅**: `util` 함수를 이용해 일관된 날짜 형식을 제공합니다.
